KBS 기안문/보고서 작성 명령어. 현재 프로젝트 폴더의 자료를 기반으로 기안문을 작성하고 현재 폴더에 저장한다.

## 인수 파싱

$ARGUMENTS 를 아래 규칙으로 파싱한다:
- `--ref "경로"` 또는 `--ref 경로`: 특정 참고문서 경로 지정 (없으면 전역 References/ 폴더 전체 참조)
- `--brief "내용"`: 인라인 브리핑 텍스트 (draft_brief.md가 없을 때 대안)
- 인수 없음: 현재 폴더 자동 스캔 + 전역 References/ 참조

예시:
- `/draft`
- `/draft --ref "/path/to/참고문서.pdf"`
- `/draft --brief "UPS 교체 공사. 예산 5천만원. 3개월 공사."`

## 실행 절차

### 1단계 — 규칙 로드

Bash로 홈 디렉토리를 확인한 뒤 `Draft_rules.md`를 Read 도구로 읽는다:

```bash
# Windows
echo $env:USERPROFILE

# Mac/Linux
echo $HOME
```

읽을 경로: `[홈경로]/.claude/skills/draft/Draft_rules.md`

### 2단계 — 소스 스캔 (현재 작업 디렉토리 기준)

현재 작업 디렉토리(CWD)를 아래 우선순위로 읽는다:

1. **draft_brief.md** 존재 여부를 먼저 확인 → 있으면 최우선 소스로 읽기
2. **`--brief "내용"` 인수** → draft_brief.md가 없고 인수가 있으면 해당 내용을 소스로 사용
3. **CWD의 문서 파일 스캔** → Glob으로 `*.md`, `*.txt`, `*.pdf` 파일 탐색 후 Read
   - `draft_brief.md`는 이미 읽었으면 중복 제외
   - 파일 수가 많으면 이름 보고 관련성 높은 것 우선 선택

### 3단계 — 참고문서 로드

- `--ref` 지정된 경우: 해당 파일만 읽기
- 미지정 시: Bash로 홈 디렉토리 확인 후 아래 경로의 References/ 폴더 전체를 Glob 후 읽기:

  ```
  [홈경로]/.claude/skills/draft/References/
  ```

### 4단계 — 기안문 작성

소스자료 + 규칙 + 참고문서를 종합하여 **HTML 파일 하나**를 **현재 작업 디렉토리(CWD)**에 저장한다.

- **파일명**: `기안문_[사업명 요약].html`
- `<!DOCTYPE>` 없이 fragment 형태로 작성
- Draft_rules.md의 HTML 스타일 규칙 전체 준수
- 표 포함 기안문 전체를 하나의 파일에 작성
- md·txt 파일은 생성하지 않는다

### 5단계 — 출처 검증표 제시 (필수)

HTML 저장 후, **클로드코드 답변(채팅)에** 기안문 본문 각 내용의 출처를 밝히는 검증표를 반드시 첨부한다. 이 문서는 **회사 공식문서**이므로 잘못된 정보가 절대 들어가서는 안 되며, 사용자가 할루시네이션(근거 없는 서술)을 직접 검수할 수 있어야 한다.

- 본문의 모든 **실질 내용**(수치·날짜·번호·고유명사·인용문구·사유·계획 등)을 아래 3분류로 표기한다:
  - **[원문]** — 소스 문서에 그대로(또는 거의 그대로) 있는 내용 → 파일명 + 위치/원문구 명시
  - **[도출]** — 소스를 근거로 바꿔 쓰거나 종합한 내용 → 어느 근거에서 나왔는지 명시
  - **[작성]** — 소스에 근거 없이 클로드가 만든 내용(일정·우선순위·정형 문구·예산 플레이스홀더 등) → **반드시 "확인 필요"로 강조**
- 출처가 불확실하면 [원문]으로 분류하지 말고 [작성/확인필요]로 둔다. **출처를 절대 지어내지 않는다.**
- 표 형식 예:

  | 항목 | 분류 | 출처 / 근거 |
  |---|---|---|
  | 접수번호 4361, 4393 | [원문] | 260604 결과알림 공문 본문 |
  | 후속 조치계획 '금년 내 시행' | [도출] | 결과보고서 제2장 권고사항 |
  | 소요예산 '별도 보고' 문구 | [작성] | 근거 없음 — 확인 필요 |

- 표 아래에 **[작성] 항목과, [도출] 중 단정적 표현(시기·주체·의무)** 만 따로 모아 "이 부분은 부서 실제 방침 확인이 필요하다"고 한 줄로 짚는다.

### 6단계 — 규칙 업데이트 제안

이번 작성에서 새로 발견한 패턴이나 규칙이 있으면 아래 형식으로 제안한다(코드펜스로 감싸지 않는다 — 이건 지금 채팅에서 사용자 승인을 구하는 텍스트이지 `Draft_rules.md`에 넣을 페이로드가 아니다).
사용자가 승인하면 `Draft_rules.md`를 즉시 업데이트한다:

`[규칙 업데이트 제안]`
추가 제안: (새 규칙 내용)
승인하시면 Draft_rules.md에 반영하겠습니다.

업데이트 대상 경로: `[홈경로]/.claude/skills/draft/Draft_rules.md`
