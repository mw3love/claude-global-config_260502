사용자 지침 변경 이력:

1. **답변 간결** (2026-07-11, 유지) — 답에서 코드·문서 변경 내역(diff)을 줄줄이 재나열하지 않는다. 결론 + 무엇을 했는지(높은 수준) + 무엇을 할지(추천)만.

2. **커밋은 확인 후** (2026-07-11 자동 도입 → 2026-07-12 되돌림) — "커밋할까요?" 묻지 않고 자동 커밋하던 규칙(구 10-a)을 되돌렸다. 검증 끝난 뒤 커밋 여부를 묻고 승인 후 진행한다.

**Why:**
ⓐ 답변 간결 — 사용자는 답변의 파일·줄 단위 변경 해설을 읽지 않는다(도구 출력에 이미 있음).
ⓑ 커밋 수동 회귀 — 자동커밋이 아낀 건 "커밋할까요?" 대화 한 번뿐이었는데, 대가로 `~/.claude`처럼 세션 내 편집이 잦은 레포에서 커밋 단위가 편집 리듬을 그대로 반영해 잘게 쪼개져 히스토리가 지저분해졌다(2026-07-11 세션의 연속 다듬기가 별개 커밋 5개로 흩어진 사례).

**How to apply:** 답변은 "diff에 보이는 걸 말로 옮기고 있나?" 싶으면 잘라낸다. 코드·설정 작업이 검증까지 끝나면 커밋 여부를 물어보고 승인 후 커밋한다. 관련: [[feedback-answer-shape]]
