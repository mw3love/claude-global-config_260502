사용자가 답변 형식에 대해 준 3가지 지침(2026-07-11):

1. **A-B-A 구조** — 결론 → 내용 → 결론. 마지막 결론에는 다음 추천·다음 세션 관련 내용을 얹는다. (기존 규칙 7 "결론부터"의 확장: 시작뿐 아니라 끝에도 결론을 둔다. 단 규칙 7 "끝의 자기 요약(diff에 이미 보이는 내용 반복) 생략"과 충돌 주의 — 마지막 결론은 *diff 재나열*이 아니라 *판단·다음 행동*을 담아야 한다.)
2. **목록 항목 사이 빈 줄** — 번호/불릿 항목 2개 이상을 붙여 쓰지 말고 한 줄 띄운다(가독성).
3. **프롬프트엔 라벨** — 복붙용 프롬프트를 낼 땐 앞줄에 `[프롬프트]` 라벨(백틱+대괄호, 2026-07-12 브라켓 규약으로 갱신)을 명시해 의견과 구별한다.

**Why:** 붙여 쓴 항목·라벨 없는 프롬프트는 터미널에서 스캔이 안 되고, 의견인지 복붙할 프롬프트인지 사용자가 되물어야 했다. A-B-A는 긴 답에서 결론을 두 번(앞·뒤) 앵커로 잡아 준다.
**How to apply:** 모든 답에 기계적으로 A-B-A를 씌우지 말 것 — 단순 확인·짧은 답엔 결론 하나면 족하다. 여러 단계·긴 답에서 앞결론/뒤결론을 세우고, 뒤결론에 다음 행동을 넣는다. 관련: [[project-rules-audit-2026-07-11]](규칙 7 answer shape).
